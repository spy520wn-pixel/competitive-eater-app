# 大胃王 APP V1 优化设计方案

## 概述

基于全面代码审查和 Critique 评审，对 V1 版本进行业务逻辑、交互体验、代码质量三个层面的优化。共 13 项改动，按优先级排列。

---

## 1. 计分算法简化 [P0]

**文件:** `src/utils/score.js`

**改动:** 去掉种类覆盖加成（`CATEGORY_BONUS`），只保留加权求和。

**新公式:**
```
总分 = Σ(分类权重 × 菜品数量)
多人就餐时: 总分 = round(总分 / 人数)
```

**权重不变:**
- 肉类/海鲜: 3
- 主食/甜点: 2
- 蔬菜/饮品/其他: 1

**同时统一分类命名（合并 #2）:** 将 `score.js` 中的 `饮料` 改为 `饮品`，新增 `蔬菜`，与 `shop-edit.vue` 保持一致。

```js
const CATEGORY_WEIGHTS = {
  '肉类': 3,
  '海鲜': 3,
  '蔬菜': 1,
  '主食': 2,
  '饮品': 1,
  '甜点': 2,
  '其他': 1
}
```

**删除:**
- `CATEGORY_BONUS` 常量
- `calculateScore` 中的 `categoryCount * CATEGORY_BONUS` 两行

**影响范围:** `cooking.vue` 的 `previewScore` 计算、`result.vue` 的最终分数展示、`record-store.js` 的经验计算。所有调用方使用 `calculateScore(items, diners)` 函数，签名不变，无需修改调用方。

## 3. 等级阈值重划 [P0]

**文件:** `src/utils/level.js`

**改动:** 替换 `LEVELS` 数组为新阈值。`scoreToExp`（score + 50）、`getLevel`、`getLevelName` 函数逻辑不变。

**新阈值（经验值）:**

| 段位 | 等级 | min | max |
|------|------|-----|-----|
| 青铜 | 1 | 0 | 99 |
| 青铜 | 2 | 100 | 299 |
| 青铜 | 3 | 300 | 599 |
| 白银 | 1 | 600 | 1199 |
| 白银 | 2 | 1200 | 1999 |
| 白银 | 3 | 2000 | 3199 |
| 黄金 | 1 | 3200 | 4999 |
| 黄金 | 2 | 5000 | 7499 |
| 黄金 | 3 | 7500 | 10999 |
| 铂金 | 1 | 11000 | 15999 |
| 铂金 | 2 | 16000 | 21999 |
| 铂金 | 3 | 22000 | 29999 |
| 钻石 | 1 | 30000 | 39999 |
| 钻石 | 2 | 40000 | 51999 |
| 钻石 | 3 | 52000 | 64999 |
| 星耀 | 1 | 65000 | 79999 |
| 星耀 | 2 | 80000 | 99999 |
| 星耀 | 3 | 100000 | 129999 |
| 王者 | — | 130000 | Infinity |

**max 值推导规则:** 每级 max = 下一级 min - 1。王者 max = Infinity。

**历史数据处理:** 不做迁移。历史分数保持原值，等级由总经验值实时计算，用户会看到等级自动更新。

## 4. 店铺照片管理 [P2]

**文件:** `src/pages/mine/shop-edit.vue`

**现状:** 照片轮播展示存在但无法上传/删除。照片来源仅为高德地图 API 导入。

**改动:**

1. **删除照片:** 轮播图上每张照片增加删除按钮（X），点击后二次确认，从 `shop.photos` 数组中移除
2. **上传照片:** 轮播区域末尾增加「+」按钮，点击弹出选择：
   - 从相册选择（`uni.chooseImage` sourceType: album）
   - 拍照（`uni.chooseImage` sourceType: camera）
3. **存储:** 选择的照片保存到本地（`uni.saveFile`），路径存入 `shop.photos`

**交互细节:**
- 删除按钮位于照片右上角，半透明红色圆形
- 「+」按钮使用虚线边框 + 加号图标，与现有照片风格统一
- 照片上限建议 9 张
- 无照片时轮播区域隐藏，不显示空容器

## 5. 结果页点餐明细样式 [P2]

**文件:** `src/pages/challenge/result.vue`

**现状:** 每个分类一个独立卡片，导致明细区域过于臃肿。

**改动:**
- 所有菜品合并到一个卡片内展示
- 按分类分组，分类名作为分组标题（小字号、半透明）
- 每行显示：菜品名 + 数量 + 单位
- 行间距适中（建议 16-20rpx），不过于宽松也不拥挤
- 默认折叠状态不变

**布局结构:**
```
┌─────────────────────────────┐
│ 点餐明细          ▼ 展开    │
├─────────────────────────────┤
│ （展开后）                  │
│ 肉类                        │
│   肥牛卷 × 3               │
│   羊肉片 × 2               │
│ 海鲜                        │
│   虾滑 × 2                 │
│   三文鱼 × 1               │
│ 主食                        │
│   炒饭 × 1                 │
└─────────────────────────────┘
```

## 6. 战绩列表排序筛选 [P2]

**文件:** `src/pages/record/record.vue`

**现状:** 店铺按最高战斗力固定排序，无法切换。

**改动:** 在列表顶部增加排序筛选栏：

**排序维度（横向标签切换）:**
- 最高战力
- 最近挑战
- 挑战次数

**排序方向:**
- 升序/降序切换按钮（箭头图标）

**默认状态:** 最近挑战 + 倒序（最近去的排最前）

**交互:**
- 筛选栏固定在搜索栏下方
- 选中的维度标签高亮（橙色）
- 切换维度自动刷新列表
- 升降序按钮点击切换，带旋转动画

## 7. 首次使用城市引导 [P1]

**文件:** `src/pages/index/index.vue`、`src/store/settings-store.js`

**现状:** 首页默认城市硬编码为「北京」。

**改动:**
1. 首次进入 APP 时（无 `defaultCity` 设置），弹出城市选择引导弹窗
2. 用户选择城市后存入 `settings-store` 的 `defaultCity` 字段
3. 后续首页直接读取 `defaultCity`，不再硬编码
4. 引导弹窗只弹一次，选择后不再出现（可在设置中修改）

**城市选择器:** 复用现有的城市列表和搜索逻辑（当前 `index.vue` 中已有城市选择弹窗）。

## 8. CSS 变量统一 [P1]

**文件:** `src/uni.scss`、`src/App.vue`

**现状:** CSS 变量在三处重复定义：
1. `uni.scss` 的 `:root` 块
2. `App.vue` 的 `:root` 块
3. `App.vue` 的 `[data-theme]` 块

**改动:**
- 保留 `App.vue` 的 `[data-theme="dark"]` 和 `[data-theme="light"]` 作为主题变量定义的唯一位置
- `uni.scss` 中删除 `:root` 块中的 CSS 变量定义，只保留 SCSS 变量（`$` 开头的）和 mixin
- `App.vue` 中删除重复的 `:root` 块，只保留 `[data-theme]` 块
- 确保所有页面的 `var(--c-xxx)` 引用都指向 `[data-theme]` 中定义的变量

**约束:必须以 APP 端为主。** 改动过程中如果 H5 和 APP 端行为冲突，优先保证 APP 端正确。改动完成后必须在 APP 端验证主题切换功能无回退。

**主题切换验证点（APP 端优先）:**
- APP 端深色→浅色：所有页面颜色正确切换
- APP 端浅色→深色：所有页面颜色正确切换
- APP 端 TabBar 颜色跟随主题
- APP 端导航栏颜色跟随主题
- APP 端页面跳转后主题保持一致
- APP 端图表组件（雷达图、柱状图、地图）主题响应正常
- H5 端主题切换（如有条件同步验证）

## 10. search-bar 修复 [P1]

**文件:** `src/components/search-bar.vue`

**改动:**

1. **清除按钮:** 输入内容后右侧显示 X 按钮，点击清空输入并立即 emit 空字符串
2. **timer 泄漏修复:** 添加 `onUnmounted` 钩子清除 pending timer

```js
import { onUnmounted } from 'vue'

let timer = null

onUnmounted(() => {
  if (timer) clearTimeout(timer)
})
```

**清除按钮交互:**
- 仅在 `modelValue` 非空时显示
- 点击后立即清空（不经防抖）
- 位置在搜索图标右侧或输入框右侧
