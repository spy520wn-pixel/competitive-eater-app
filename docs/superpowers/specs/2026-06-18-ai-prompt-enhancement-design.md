# AI 创作提示词增强设计文档

## 背景

当前 AI 创作功能的提示词过于简单，只列出菜品分类汇总（如"肉类3、海鲜2"），导致生成的图片质量差，无法体现这次自助餐的具体内容。

## 目标

改进提示词构建逻辑，让生成的图片更能体现用户实际吃的内容，提升图片质量和个性化程度。

## 设计方案

### 核心思路

两种创作模式（文生图、图生图）都通过 LLM 生成高质量提示词，而不是使用简单的本地模板。

### 流程设计

#### 文生图模式

```
战绩数据 → LLM 生成提示词 → 图像生成 API → 返回图片 URL
```

**输入数据：**
- 菜品列表（名称、数量、分类）
- 店铺名称、档位名称
- 风格选择
- 战斗力评分、时长

**LLM 任务：**
- 分析菜品列表，提取亮点菜品
- 生成画面感强的场景描述
- 结合风格特点，输出专业级提示词

**输出示例：**
```
在海底捞火锅的热闹氛围中，一位大胃王正在享用丰盛的美食。
桌上摆满了肥牛卷、虾滑、毛肚、羊肉卷等8道美味，
搭配冰淇淋和饮料作为完美收尾。
战斗力评分85分，用时90分钟。
搞笑夸张风格，食物放大变大，添加表情包元素，色彩鲜艳，有趣幽默。
```

#### 图生图模式

```
照片 + 战绩数据 → LLM 识别+生成提示词 → 图像生成 API（含参考图）→ 返回图片 URL
```

**输入数据：**
- 用户上传的照片（1-3张）
- 菜品列表（名称、数量、分类）
- 店铺名称、档位名称
- 风格选择
- 战斗力评分、时长

**LLM 任务：**
- 识别照片中的食物内容
- 结合战绩数据，补充照片中未体现的菜品
- 生成融合照片内容和战绩数据的提示词
- 应用所选风格

**输出示例：**
```
参考图片中的火锅场景，桌上摆满了各种美食。
结合战绩数据，补充了照片中可能未体现的菜品：
肥牛卷、虾滑、毛肚、羊肉卷等8道美味。
在海底捞火锅的氛围下，大胃王挑战成功。
搞笑夸张风格，保留原图构图，增加夸张表现元素。
```

### 技术实现

#### 配置复用

使用现有的识图大模型配置：
- `ocrServiceUrl` - LLM 服务地址
- `ocrApiKey` - API Key
- `ocrModel` - 模型名称（默认 'agnes-2.0-flash'）

#### 新增函数

**1. `generatePrompt(record, style)`**
- 用途：文生图模式生成提示词
- 输入：战绩记录、风格名称
- 输出：高质量提示词字符串

**2. `generatePromptWithPhoto(record, style, photoPaths)`**
- 用途：图生图模式生成提示词
- 输入：战绩记录、风格名称、照片路径数组
- 输出：高质量提示词字符串

#### 修改函数

**`generateImage({ record, style, photoPaths })`**
- 调用 `generatePrompt` 或 `generatePromptWithPhoto` 获取提示词
- 使用新提示词调用图像生成 API

#### 错误处理

- LLM 调用失败时，降级使用本地模板（现有的简单提示词）
- 保留现有的重试机制（withRetry）
- 图像生成失败时，提供重试选项

### LLM Prompt 设计

#### 文生图 System Prompt

```
你是一个专业的 AI 创作提示词生成助手。你的任务是根据用户的大胃王挑战战绩数据，生成高质量的图像生成提示词。

## 输入数据
- 店铺名称
- 档位名称（可选）
- 菜品列表（名称、数量、分类）
- 风格选择
- 战斗力评分
- 时长

## 输出要求
1. 生成一段画面感强的场景描述
2. 突出具体菜品名称，而非分类汇总
3. 结合风格特点，添加相应的视觉元素
4. 适合生成朋友圈炫耀的图片
5. 长度控制在 200-300 字

## 示例输入
- 店铺：海底捞火锅
- 菜品：肥牛卷x3、虾滑x2、毛肚x1、羊肉卷x2、冰淇淋x1
- 风格：搞笑夸张
- 战斗力：85分
- 时长：90分钟

## 示例输出
在海底捞火锅的热闹氛围中，一位大胃王正在享用丰盛的美食。桌上摆满了肥牛卷、虾滑、毛肚、羊肉卷等8道美味，搭配冰淇淋作为完美收尾。战斗力评分85分，用时90分钟。搞笑夸张风格，食物放大变大，添加表情包元素，色彩鲜艳，有趣幽默。
```

#### 图生图 System Prompt

```
你是一个专业的 AI 创作提示词生成助手。你的任务是根据用户上传的照片和大胃王挑战战绩数据，生成高质量的图像生成提示词。

## 输入数据
- 用户上传的照片（将作为参考图）
- 店铺名称
- 档位名称（可选）
- 菜品列表（名称、数量、分类）
- 风格选择
- 战斗力评分
- 时长

## 输出要求
1. 先描述照片中的食物内容
2. 结合战绩数据，补充照片中可能未体现的菜品
3. 生成融合照片内容和战绩数据的场景描述
4. 结合风格特点，添加相应的视觉元素
5. 提示词中注明"参考图片中的内容"
6. 长度控制在 200-300 字

## 示例输入
- 照片：火锅照片
- 店铺：海底捞火锅
- 菜品：肥牛卷x3、虾滑x2、毛肚x1、羊肉卷x2、冰淇淋x1
- 风格：搞笑夸张
- 战斗力：85分
- 时长：90分钟

## 示例输出
参考图片中的火锅场景，桌上摆满了各种美食。结合战绩数据，补充了照片中可能未体现的菜品：肥牛卷、虾滑、毛肚、羊肉卷等8道美味，搭配冰淇淋作为完美收尾。在海底捞火锅的氛围下，大胃王挑战成功，战斗力评分85分，用时90分钟。搞笑夸张风格，保留原图构图，增加夸张表现元素，食物放大变大，添加表情包元素。
```

### 数据流

```
用户选择战绩 → 选择照片（可选）→ 选择类型 → 选择风格
                                    ↓
                            调用 generateImage
                                    ↓
                    ┌───────────────┴───────────────┐
                    ↓                               ↓
            文生图模式                        图生图模式
                    ↓                               ↓
        generatePrompt()              generatePromptWithPhoto()
                    ↓                               ↓
            LLM 生成提示词              LLM 识别照片+生成提示词
                    ↓                               ↓
                    └───────────────┬───────────────┘
                                    ↓
                        图像生成 API（含参考图）
                                    ↓
                            返回图片 URL
```

### 实现步骤

1. **新增 LLM 提示词生成函数**
   - `generatePrompt(record, style)`
   - `generatePromptWithPhoto(record, style, photoPaths)`

2. **修改 `generateImage` 函数**
   - 调用新函数获取提示词
   - 保留降级逻辑

3. **测试验证**
   - 文生图模式测试
   - 图生图模式测试
   - 错误处理测试

### 预期效果

- 图片更能体现用户实际吃的内容
- 提示词质量提升，生成效果更好
- 保留降级机制，确保稳定性
