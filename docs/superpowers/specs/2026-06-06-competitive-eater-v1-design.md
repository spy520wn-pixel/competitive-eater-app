# 大胃王APP V1 设计方案

## 1. 产品定位

V1 为个人使用的"大胃王战绩记录工具"，核心价值是让每次自助餐体验可视化、可量化、可炫耀。

- **目标用户**：自助餐爱好者、大胃王挑战参与者
- **产品形态**：UniApp 多端应用（H5 / APP / 小程序）
- **数据存储**：纯本地存储（uni.storage），无需后端服务器

## 2. 核心用户流程

```
准备阶段                    就餐阶段                    结算阶段
─────────                  ─────────                  ─────────
选择/新增店铺               启动倒计时                  自动生成：
  ↓                          ↓                          ├─ 分类食量统计图
录入店铺菜单                 从菜单选菜（点外卖式）        ├─ 战绩小票（Canvas）
  ├─ 菜品名称                 ├─ 左侧分类Tab              ├─ AI创意海报（可选）
  ├─ 所属分类                 ├─ 右侧菜品列表              └─ 大胃王等级更新
  ├─ 计量单位                 ├─ 加减数量                  
  └─ 店铺就餐时限              └─ 实时累计                   ↓
                               ↓                       分享/保存
可选：图片识别导入菜单        提前结束 或                  ↓
                             倒计时到自动锁定             存入历史战绩
```

## 3. 页面结构

| 页面 | 功能 |
|------|------|
| **首页** | 数据看板（统计图/雷达图 + 大胃王等级）+ 地图足迹 |
| **战绩** | 按店铺列表展示最高战绩，点开查看全部记录 |
| **挑战** | 就餐记录主流程（选店铺 → 点菜 → 倒计时） |
| **我的** | 个人信息、店铺管理（新增/编辑/菜单维护/图片识别导入）、数据备份、设置 |

## 4. 数据模型

### Shop（店铺）

```javascript
{
  id: string,              // 唯一标识
  name: string,            // 店铺名称
  address: string,         // 地址
  category: string,        // 分类：自助餐/火锅/烧烤/其他
  city: string,            // 城市，用于地图筛选
  location: {              // 经纬度，用于地图标注
    lng: number,
    lat: number
  },
  mealTimeLimit: number,   // 就餐时限（分钟）
  hasTiers: boolean,       // 是否分档位
  tiers: Tier[],           // 档位列表（hasTiers=true 时有值）
  menu: MenuItem[],        // 菜单列表（hasTiers=false 时使用）
  createdAt: datetime
}
```

### Tier（档位）

```javascript
{
  id: string,
  shopId: string,
  name: string,            // 档位名称，如：基础档、豪华档、尊享档
  menu: MenuItem[]         // 该档位的菜单
}
```

### MenuItem（菜单项）

```javascript
{
  id: string,
  shopId: string,
  tierId: string,          // 所属档位ID（无档位时为空）
  name: string,            // 菜品名称
  category: string,        // 分类：肉类/海鲜/主食/甜点/饮料/其他
  unit: string             // 单位：盘/份/只/杯/个
}
```

### Record（战绩记录）

```javascript
{
  id: string,
  shopId: string,
  tierId: string,          // 档位ID（无档位时为空）
  tierName: string,        // 档位名称（冗余，方便展示）
  startTime: datetime,     // 开始时间
  endTime: datetime,       // 结束时间
  duration: number,        // 实际就餐时长（分钟）
  status: string,          // 进行中/已完成
  items: RecordItem[],     // 点餐明细
  score: number,           // 战斗力评分
  createdAt: datetime
}
```

### RecordItem（点餐明细）

```javascript
{
  menuItemId: string,      // 关联菜单项
  name: string,
  category: string,
  quantity: number,
  unit: string
}
```

### 倒计时规则

- 从选择「开始挑战」起计时
- 倒计时归零 → 自动锁定，禁止新增/修改点餐
- 倒计时 < 5 分钟时，顶部变红提醒
- 可提前手动结束，不可延时

### 点菜交互

- 每个菜品一行：名称（左）+ 单位标签 + 数量加减器（右）
- 点「+」自动 +1，长按可输入精确数量（支持小数如 0.5）
- 底部实时显示：已选 X 道菜，覆盖 Y 个分类

## 6. 评分与等级体系

### 战斗力评分算法

按分类分别统计食量，加权汇总，体现"总量大 + 种类多"的综合实力。

```
总战斗力 = Σ（每个分类的分类得分） + 种类丰富度加成

分类得分 = 该分类下所有菜品的（数量 × 分类权重）
```

**分类权重：**

| 分类 | 权重 | 说明 |
|------|------|------|
| 肉类 | 3分/单位 | 高蛋白，更能体现食量 |
| 海鲜 | 3分/单位 | 同上 |
| 主食 | 2分/单位 | |
| 甜点 | 2分/单位 | |
| 饮料 | 1分/单位 | |
| 其他 | 1分/单位 | |

**种类丰富度加成：** 每覆盖一个分类 +20 分

**计算示例：**
> 吃了 5 盘肉（5×3=15）、3 盘海鲜（3×3=9）、2 盘主食（2×2=4）、1 杯饮料（1×1=1），覆盖 4 个分类（4×20=80）
>
> 总战斗力 = 15 + 9 + 4 + 1 + 80 = **109 分**

### 大胃王等级体系

游戏排位制，取用户所有战绩中的**单次最高战斗力**作为等级依据。

| 等级 | 名称 | 战斗力区间 |
|------|------|-----------|
| Ⅰ | 青铜 | 0 - 49 |
| Ⅱ | 白银 | 50 - 99 |
| Ⅲ | 黄金 | 100 - 199 |
| Ⅳ | 铂金 | 200 - 349 |
| Ⅴ | 钻石 | 350 - 499 |
| Ⅵ | 星耀 | 500 - 799 |
| Ⅶ | 王者 | 800+ |

### 二、AI 创意海报（调用 AI API）

**生成流程：**

```
选择风格模板
    │
    ├─ 搞笑夸张风（食物放大、表情包元素）
    ├─ 动漫二次元风
    ├─ 霸气宣言风（大字报、奖杯元素）
    └─ 简约战绩风（小票+照片融合）
    │
    ↓
可选：上传 1~3 张实际就餐照片
    │
    ↓
系统自动构建 Prompt（战绩数据 + 风格描述 + 照片）
    │
    ↓
调用 AI 生图 API
    │
    ↓
生成预览（支持重新生成）
    │
    ↓
保存 / 分享
```

**Prompt 构建策略：**
- 系统根据战绩数据自动生成结构化 Prompt
- 包含：店铺名、核心菜品、战斗力、等级
- 风格关键词由模板预设
- 若上传照片，附加图片作为参考输入

## 12. 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端框架 | UniApp + Vue 3 | 一套代码多端运行（H5/APP/小程序） |
| 本地存储 | uni.storage | 跨平台本地存储，无需后端 |
| 地图 | 高德地图 JS API | 免费额度够用，地图展示+标注+点击交互 |
| 小票生成 | Canvas 模板 | 前端绘制，无网络依赖 |
| AI 生图 | 通义千问 / 文心一言 API | 多风格创意海报生成 |
| AI 菜单识别 | OCR API（视觉模型） | 图片识别导入菜单 |
